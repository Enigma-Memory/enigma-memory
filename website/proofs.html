Proofs

Proof without publishing your memory.

Enigma proof artifacts can show hashes, roots, counts, timestamps, signatures, opaque references, and validation results without exposing raw memory text.

View proof activity

What a proof can show

  • A memory create, retrieval, context-pack, update, or tombstone event produced a receipt.
  • The receipt verifies offline against Enigma-controlled vault state.
  • Exported artifacts contain commitments and metadata, not raw memory plaintext.

What a proof must never contain

  • Raw memory text, prompts, transcripts, completions, or embeddings.
  • Provider responses, account IDs, tenant names, or credentials.
  • Local absolute paths or private keys.

How unsafe artifacts are rejected

Enigma scans exports for plaintext-like fields before producing a public-safe proof. If private data is detected, the export is rejected with guidance on what to remove. Public examples use synthetic opaque refs only, such as ref_abc123.

What local proof is not

Local proof is not provider deletion or model forgetting. It shows what Enigma did in its own vault. Solana-ready means prepared for review, not submitted on-chain unless separate transaction evidence exists.

Boundary note: Enigma proves Enigma-controlled local vault and receipt events. It does not prove that a closed provider deleted hidden copies or that a model forgot training, fine-tuning, cache, or personalization state.
Advanced proof details

Advanced proof details include schemas, validators, signatures, roots, nullifiers, and Solana-ready payload language. These are available in the source checkout and API reference for developers and auditors.
