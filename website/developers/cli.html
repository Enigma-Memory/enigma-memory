Developers

CLI and MCP setup for power users.

Use npm, Node, terminal commands, and manual MCP snippets only when you want the developer engine instead of the desktop-first consumer app.

This is not the default consumer setup path. Most people should use the desktop app.

Prerequisites

  • Node.js 24 or later.
  • A local filesystem path for the Enigma vault bundle.
  • No database, provider credential, cloud credential, or hosted Enigma account for the local path.

Install with npm

npm install -g enigma-memory

Initialize a local vault, verify it, then preview client changes before writing anything:

enigma quickstart --bundle ./.enigma/bundle.json
enigma doctor --bundle ./.enigma/bundle.json
enigma drive health --bundle ./.enigma/bundle.json
enigma connect claude-desktop --bundle ./.enigma/bundle.json --dry-run

After reviewing a dry-run, connect one intended client at a time:

enigma connect claude-desktop --bundle ./.enigma/bundle.json
enigma connect cursor --bundle ./.enigma/bundle.json --dry-run
enigma connect kimi-code --bundle ./.enigma/bundle.json --dry-run

Run the MCP server

Start the Enigma MCP stdio server:

ENIGMA_BUNDLE="$HOME/.enigma/bundle.json" enigma-mcp

Or through the CLI:

ENIGMA_BUNDLE="$HOME/.enigma/bundle.json" enigma mcp serve

Generic MCP client entry

Use an absolute bundle path and set ENIGMA_BUNDLE directly in the entry for portability:

{
  "mcpServers": {
    "enigma": {
      "command": "enigma-mcp",
      "args": [],
      "env": {
        "ENIGMA_BUNDLE": "/absolute/path/to/.enigma/bundle.json"
      }
    }
  }
}

If a GUI app cannot find shell-installed binaries, use --mcp-command with an absolute path to the executable.

Export and verify proof artifacts

enigma export --bundle ./.enigma/bundle.json --out ./.enigma/export.json
enigma verify --export ./.enigma/export.json

Exported proof artifacts contain commitments, roots, and receipt metadata. Do not paste raw memory plaintext into relay records, witness checkpoints, SIEM events, public proof artifacts, or shell command lines.

Source checkout

Use a source checkout only when you need source-only docs, Docker assets, browser-extension scaffolding, package development, or release scripts.

git clone https://github.com/Enigma-Memory/enigma-memory.git
cd enigma-memory
npm run install:local -- --execute --init-vault --bundle ./.enigma/bundle.json
enigma doctor --bundle ./.enigma/bundle.json

Full installation and connector details are in docs/install-anywhere.md and docs/client-connectors.md.

Claim boundary

Enigma proves Enigma-controlled local vault and receipt events. It does not prove provider deletion, provider-native memory removal, model forgetting, hosted SaaS readiness, compliance certification, or benchmark superiority.
