Install guide

Install Enigma Memory when your platform is marked ready.

Signed Windows and macOS installers are still in progress. General consumers should wait until the download page marks their platform as signed and ready; testers can use labeled development builds only when they intentionally accept that boundary.

Desktop install

macOS

  1. Open the download page and confirm macOS is marked signed, notarized, and ready for public install.
  2. Download only from the official Enigma download channel after that status is ready.
  3. Open the disk image or package, then drag Enigma Memory into Applications or run the installer package.
  4. Open Enigma Memory. The first-run setup creates your local vault.

Windows

  1. Open the download page and confirm Windows is marked signed and ready for public install.
  2. Download only from the official Enigma download channel after that status is ready.
  3. If Windows SmartScreen appears, cancel the prompt and check the launch status. General consumers should wait for a signed build marked ready for public install.
  4. Open Enigma Memory after the installer completes. The first-run setup creates your local vault.

Linux

  1. Download the AppImage or package from the download page.
  2. Mark the AppImage executable, or install the package with your distribution tools.
  3. Open Enigma Memory. The first-run setup creates your local vault.
Boundary note: The desktop app bundles its runtime. Consumers should not install Node or npm for the default path.

If your system blocks the installer

Operating systems sometimes warn about apps downloaded from the web. Do not bypass those warnings by default.

  • Recommended for consumers: Stop, check the launch status, and wait for a signed build marked ready for public install.
  • Testing-only: If you are validating a pre-launch build from an official Enigma channel, use the OS-specific steps in the installer blocked article.
  • Linux: Check the package signature or checksum with your distribution tools before opening the package.

Never continue past a warning for installers from unknown sources, unofficial mirrors, email attachments, or copied links.

Install from npm or source

This is the advanced developer path. For the default consumer experience, use the desktop app above.

If you want the command-line engine instead, see the CLI appendix for npm install, Node requirements, setup commands, and source checkout.
