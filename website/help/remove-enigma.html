Safe removal

Remove Enigma without losing your local vault by accident.

Disconnect an AI app first, then uninstall the desktop app only when you intend to remove the app itself. Your Memory Drive should be treated as user data and kept unless you choose a separate destructive removal path.

Remove Enigma from one connected app

  1. Open Enigma Memory.
  2. Open the connected apps or Memory Drive dashboard.
  3. Select the app you want to remove from Enigma.
  4. Choose Disconnect or the app-specific removal action.
  5. Review the preview. Enigma should remove only its own connector entry and keep unrelated app settings intact.
  6. Restart the AI app if Enigma asks you to.
  7. Run the app connection test or health check again.
Boundary note: Disconnecting Enigma from an app affects Enigma-controlled local connector setup only. It does not delete the provider's own logs, caches, account history, or provider-native memory.

Uninstall the desktop app

Windows

Use Windows Settings to uninstall the app package when signed installers are available. The normal uninstall path should remove app binaries and leave your local Memory Drive data alone.

macOS

Remove the app from Applications when signed/notarized builds are available. Treat your local Memory Drive as separate user data unless you explicitly choose full removal.

Before reinstall

Reopen Enigma after reinstall and let it detect the existing Memory Drive. Do not delete local vault data to fix a normal app update or reinstall issue.

Full removal is separate and destructive.

Only remove the local Memory Drive when you intentionally want to delete Enigma-owned local memory data. This should be a separate explicit action, not part of the normal uninstall path.

  • Export or back up anything you want to keep first.
  • Disconnect AI apps before destructive local data removal.
  • Do not send raw vault files, full logs, app configs, or local paths to support.
  • If you are unsure, use safe support sharing and send only the support code or public-safe summary.

If removal fails

Use the dashboard repair or support summary first. Support should ask for the public-safe support code, not raw memory, local paths, complete MCP JSON, screenshots of provider conversations, or full logs.
