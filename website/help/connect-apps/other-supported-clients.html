Connect apps

Connect Kimi Code, Cline, Roo Code, OpenCode, or Generic MCP.

Use Enigma Memory's desktop Connections screen first. Manual MCP JSON is an advanced fallback, not the consumer setup path.

Supported client paths

Kimi Code

Open Kimi Code once, then return to Enigma and refresh detection. Approve the previewed MCP entry and restart Kimi Code if prompted.

VS Code / Cline

Install and open the Cline extension first. Enigma detects the Cline settings area, previews the change, and asks you to reload VS Code when needed.

Roo Code

Open Roo Code once so its settings exist. Enigma previews the Roo MCP change and asks for approval before writing anything.

OpenCode

Open OpenCode once, refresh detection in Enigma, approve the preview, then restart or reload OpenCode if Enigma asks you to.

Generic MCP

Use this only for MCP-compatible clients without a dedicated card. Enigma should still show a preview before it writes a selected configuration.

Connection steps

  1. Install the AI client and open it at least once.
  2. Open Enigma Memory and go to Connections.
  3. Click Refresh detection.
  4. Choose the detected client and click Connect.
  5. Review the human-readable preview. Approve only if it names the client you intended to connect.
  6. Restart or reload the client if Enigma asks you to.
Boundary note: Enigma connectors configure the client to start Enigma's local MCP server. They do not delete provider logs, erase model behavior, or change provider account settings.

If the client is not detected

  • Confirm the client is installed and has been opened once.
  • Close and reopen Enigma Memory, then refresh detection again.
  • Check the AI app not detected guide for common blockers.
Manual MCP configuration

Manual MCP JSON is for developers and troubleshooting only. If the desktop app cannot detect or connect a supported client, see the CLI appendix for manual snippets and connector commands.
