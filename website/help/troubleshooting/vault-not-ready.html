Troubleshooting

Vault not ready.

Enigma cannot find or open your local Memory Drive. Start with the repair buttons inside the desktop app; command-line repair is an advanced fallback at the bottom of this page.

Symptom

The health dashboard shows a Memory Drive warning, or the setup wizard cannot finish creating your local drive.

Likely causes

  • Setup closed before the Memory Drive was created.
  • The drive or folder that holds Enigma’s data is not available right now.
  • A system permission is stopping Enigma from opening its own folder.

Try this first

  1. Open Enigma Memory and choose Create local vault in the setup wizard.
  2. If you already created one, choose Repair vault on the health dashboard.
  3. If you moved the vault folder, make sure that drive or folder is connected and available.

If that does not work

  1. Restart Enigma Memory and check the health dashboard again.
  2. If a backup exists, choose Restore from backup in the recovery prompt.
  3. If no backup exists and Enigma still cannot open the vault, choose Safe reset. This starts a fresh Enigma vault and does not change Claude, Cursor, or other connected app settings.

What to send support

Safe to share: health status, app version, support code, and whether the vault was created before the warning appeared. Do not send raw memory, prompts, transcripts, credentials, account IDs, or local paths. See what is safe to share with support.

Advanced command-line repair

If the desktop app repair options do not resolve the issue, see the CLI appendix for terminal-based repair commands. Run these only if you are comfortable with the command line.
