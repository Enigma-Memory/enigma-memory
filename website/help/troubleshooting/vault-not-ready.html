Troubleshooting

Vault not ready.

Enigma cannot find or read your local vault. Use the in-app repair first; CLI repair is available only in the advanced panel.

Symptom

The health dashboard shows a warning for the Memory Drive, or the setup wizard cannot create the vault.

Likely causes

  • First run was interrupted before the vault was created.
  • The vault location is not readable or writable.
  • An app or system setting is blocking Enigma from accessing its app data folder.

Try this first

  1. Open Enigma Memory and click Create local vault in the setup wizard.
  2. If the vault already exists, click Repair vault on the health dashboard.
  3. If you changed the vault location, make sure the new location is available and writable.

If that does not work

  1. Restart Enigma Memory and check the health dashboard again.
  2. If a backup exists, choose Restore from backup in the recovery prompt.
  3. If no backup exists and the vault is corrupted, choose Safe reset. This removes only Enigma app config and recreates the vault; it does not touch connected app settings.

What to send support

Safe to share: health status, app version, support code, and whether the vault was created before the warning appeared. Do not send raw memory, prompts, transcripts, credentials, account IDs, or local paths.

CLI repair commands

If the desktop app repair options do not resolve the issue, see the CLI appendix for terminal-based repair commands. Run these only if you are comfortable with the command line.
