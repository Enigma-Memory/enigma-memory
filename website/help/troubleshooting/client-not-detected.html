Troubleshooting

AI app not detected.

Make sure the app is installed and opened, then refresh detection in Enigma. Manual configuration is available only in the advanced panel.

Symptom

Enigma Memory does not list a supported AI app during client detection, or the app status shows Not installed when you expect it to be found.

Likely causes

  • The app was installed but never opened, so its config folder does not exist yet.
  • The app was installed in a location Enigma cannot read.
  • The app is not a supported client.

Try this first

  1. Make sure the app is installed.
  2. Open the app at least once and close it.
  3. Return to Enigma Memory and click Refresh detection.
  4. If the app appears, click Connect and approve the previewed change.

If that does not work

  1. Restart the AI app and Enigma Memory, then refresh detection again.
  2. Check that you are using a supported client version.
  3. If the app is supported but still not detected, use the manual MCP JSON in the advanced panel.

What to send support

Safe to share: app version, OS family and version, the connector ID shown in Enigma, and the visible support code. Do not send raw memory, prompts, transcripts, credentials, account IDs, or complete client config files.

Manual MCP configuration

Manual MCP JSON is for developers and troubleshooting only. If the desktop app cannot detect a supported client, see the CLI appendix for manual snippets and connector commands.
