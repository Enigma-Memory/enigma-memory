Troubleshooting

Installer blocked by the operating system.

Operating systems sometimes warn about apps downloaded from the web. General consumers should wait for a signed build marked ready for public install instead of bypassing the warning.

Symptom

You see a security warning when you try to open the Enigma Memory installer, or the installer is moved to trash/quarantine.

Windows SmartScreen

SmartScreen can appear when Windows does not yet trust a downloaded installer.

  • Recommended for consumers: Cancel the prompt, check the launch status, and wait for a signed build marked ready for public install.
  • Testing-only: If you are validating a pre-launch build from an official Enigma channel, click More info, then Run anyway.

If the warning names an unknown publisher or source, stop and download again from the official Enigma page instead of continuing.

macOS Gatekeeper

Gatekeeper can appear when macOS cannot verify a downloaded app.

  • Recommended for consumers: Stop, check the launch status, and wait for a signed build marked ready for public install.
  • Testing-only: If you are validating a pre-launch build from an official Enigma channel, right-click the installer or app, choose Open, then confirm.

If Gatekeeper says the app cannot be verified and you are not testing an official pre-launch build, wait for the launch status to show a public-install-ready build.

If that does not work

  • Check the launch status for the current public-install readiness.
  • Download the installer again from the official download page.
  • Do not run installers from unofficial mirrors or email attachments.

What to send support

Safe to share: app version, OS family and version, installer channel, and the visible support code. Do not send raw memory, prompts, transcripts, credentials, account IDs, or local paths. See what is safe to share with support.

Trust prompt details

Public trust setup can vary by release channel. If you are testing a pre-launch build, an operating-system trust prompt may be expected. See the CLI appendix only if you intentionally want the developer engine path.
