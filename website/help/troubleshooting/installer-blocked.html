Troubleshooting

Installer blocked by the operating system.

Operating systems sometimes warn about apps downloaded from the web. Use signing-safe guidance and do not bypass warnings for unknown sources.

Symptom

You see a security warning when you try to open the Enigma Memory installer, or the installer is moved to trash/quarantine.

Windows SmartScreen

  1. Click More info on the SmartScreen prompt.
  2. Click Run anyway, but only if the installer came from an Enigma distribution channel.

For public launch, Enigma uses a signed Windows installer or Store/MSIX package to reduce SmartScreen friction.

macOS Gatekeeper

  1. Right-click the installer or app and choose Open.
  2. Click Open in the dialog to confirm, but only if the installer came from an Enigma distribution channel.

For public launch, Enigma uses Developer ID signing, notarization, and stapling so Gatekeeper recognizes the app.

If that does not work

  • Download the installer again from the official download page.
  • Check that the file size and name match the download page.
  • Do not run installers from unofficial mirrors or email attachments.

What to send support

Safe to share: app version, OS family and version, installer channel, and the visible support code. Do not send raw memory, prompts, transcripts, credentials, account IDs, or local paths.

Signing technical details

Public launch builds are signed with the chosen production trust channel. If you are testing a pre-launch build, the trust prompt may be expected. See the CLI appendix for the developer engine path.
