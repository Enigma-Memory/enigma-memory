Install guide

Install Enigma Memory on macOS when signing is ready.

General consumers should wait until macOS is marked signed, notarized, stapled, and ready on the Enigma download page. No terminal or Node install is required for the signed desktop path.

macOS install steps

  1. Confirm the download page marks macOS as signed, notarized, stapled, and ready for public install.
  2. Download the macOS installer only from the official Enigma download channel after that status is ready.
  3. Open the disk image or package, then follow the prompts or drag Enigma Memory into Applications.
  4. If macOS says the app cannot be verified, stop and check the launch status.
  5. Open Enigma Memory from Applications after install. The first-run wizard creates your Memory Drive.
Boundary note: The desktop app bundles its runtime. Consumers should not install Node or npm for the default path.

If macOS blocks the installer

macOS Gatekeeper may warn about apps downloaded from the web. Do not continue past it by default.

  • Recommended for consumers: Stop, check the launch status, and wait for a signed build marked ready for public install.
  • Testing-only: If you are validating a pre-launch build from an official Enigma channel, right-click the installer or app, choose Open, then confirm.

Do not bypass security warnings for unknown publishers, unofficial mirrors, email attachments, or copied links. See the installer blocked article for more details.

Install from npm or source

This is the advanced developer path. For the default consumer experience, use the desktop app above.

If you want the command-line engine instead, see the CLI appendix for npm install, Node requirements, setup commands, and source checkout.
