Troubleshooting

Fix common setup issues from the health dashboard.

Enigma explains what is wrong, what it can fix automatically, and what you need to approve.

Common issues

Installer blocked by the operating system

Try this first: Cancel the warning, check the launch status, and wait for a signed build marked ready for public install.

What Enigma checks: Whether the installer came from an expected Enigma distribution channel for your release.

If you are testing: Use bypass steps only for official Enigma pre-launch builds. Do not bypass warnings for unknown sources.

Vault not ready

Try this first: Open Enigma and click Create Memory Drive in the setup wizard.

What Enigma checks: The local Memory Drive exists and is readable.

If that does not work: Use the in-app Repair vault option. CLI repair is available in the advanced panel.

AI app not detected

Try this first: Make sure the app is installed and has been opened at least once, then click Refresh detection in Enigma.

What Enigma checks: Whether the app has created its settings area and whether Enigma can preview a safe connection change.

If that does not work: Restart the AI app, or use advanced manual settings only from the troubleshooting panel.

Proof export rejected as unsafe

Try this first: Remove any raw memory text, prompts, transcripts, or provider responses from the export fields.

What Enigma checks: The export contains only hashes, roots, opaque refs, counts, timestamps, signatures, and validation results.

If that does not work: Use the in-app proof activity screen, which exports safe artifacts automatically.

Send safe diagnostics to support

Safe to share: Health status, app version, sanitized error codes, and public-safe proof refs.

Do not share: Raw memory, prompts, transcripts, credentials, account IDs, or local paths. See the safe support sharing guide before sending screenshots or copied diagnostics.

CLI repair commands

If the desktop app repair options do not resolve the issue, see the CLI appendix for terminal-based repair commands. Run these only if you are comfortable with the command line.
