Local-first memory for the AI apps you already use

Your AI memory, kept in your own local vault.

Start with the desktop app and first-run wizard. Enigma creates a local vault, previews supported app connections, and avoids Node, terminal, or JSON setup on the default path.

Public launch signing is still tracked openly. Use signed desktop builds when available; current unsigned builds are labeled for testing and development, not general consumer install.

How you use it

1. Download the desktop app

Start on the download page. When signed installers are not available, unsigned development builds are labeled before you choose them.

2. Create your Memory Drive

First run creates a private local vault for Enigma Memory data on this computer.

3. Connect an AI app

Choose a supported app, preview the connection, and let that app ask Enigma for memory only when you allow that use.

What Enigma does

One local vault

Enigma keeps its canonical memory vault on your device. No cloud account or provider login is required for the default desktop path.

Connect your assistants

Detect installed AI clients, preview the setting Enigma will change, and approve the connection. Claude Desktop, Cursor, Kimi Code, and more.

Health and proof

Check vault and connection readiness from the health dashboard. Export public-safe proof artifacts without exposing raw memory text.
