Public launch

Launch status and code-signing timeline.

The desktop-first flow is the public-launch target, but general consumer release is still blocked by signed Windows and macOS installers. Azure Artifact Signing now requires an eligible paid subscription before Public Trust setup can continue; Apple notarization is not complete.

This page is public-safe: private account details, restricted signing material, and internal locations are omitted.

Current status

Tauri updater key

Configured

Configured. The updater signing key has a protected repository custody path and the matching public key is committed in the desktop app configuration, so future auto-updates can be verified after release signing is complete.

Azure subscription

Upgrade required

Blocked. Azure Portal rejected the current subscription for Artifact Signing because free, trial, and sponsored subscriptions are not eligible. Upgrade or switch to an eligible paid subscription before creating the signing account.

Azure Artifact Signing / Public Trust

Blocked

Blocked before account creation. After an eligible paid subscription is available, the Artifact Signing account, Public Trust identity validation, and certificate profile still need to be completed before trusted Windows signing can be claimed.

Apple Developer enrollment

Pending

Pending. Enrollment in the Apple Developer Program is required before macOS Developer ID certificates, signing, and notarization can be completed.

Expected timeline

  1. 1

    Windows signed installer

    Blocked on Azure subscription eligibility first, then Artifact Signing identity validation. Azure rejected the current subscription as free/trial/sponsored; after an eligible paid subscription is available, Public Trust validation typically takes minutes to hours and requires a mobile device for document and selfie capture.

  2. 2

    macOS signed and notarized installer

    Blocked on Apple Developer Program enrollment. After enrollment, Developer ID Application and Installer certificates must be created, exported, and added to the protected release-signing environment; then CI can sign, notarize, and staple the macOS build. Enrollment review usually takes 1–5 business days for individuals and 1–4 weeks for organizations.

  3. 3

    General consumer release

    Once both signed installers are available, the public download page will switch to the signed builds and the launch will be considered ready for general consumers.

What you can do today

Until signed installers are ready, you have two paths:

  • Wait for the signed build: This is the recommended consumer path. The download page will switch to signed Windows and macOS installers only after signing evidence exists.
  • Use current development builds: Unsigned builds may be available from official Enigma release channels for testing and development. They are not the general consumer install, and your operating system may show a security warning because the binaries are not yet signed.

Do not bypass security warnings for installers from sources you do not trust.
