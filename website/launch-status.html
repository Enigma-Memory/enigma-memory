Public launch

Launch status and code-signing timeline.

Enigma Memory is ready for daily use. The remaining public-launch blocker is distributing signed desktop installers for Windows and macOS. This page tracks where we are and what comes next.

See current status Go to downloads

This page is public-safe: no account IDs, secrets, or internal paths are shown.

Current status

Tauri updater key

Configured. The updater signing key is stored as a repository secret and the matching public key is committed in the desktop app configuration, so future auto-updates can be verified.

Azure subscription

Active. A paid Azure subscription is available for Artifact Signing. Identity validation is the next step before a trusted signing certificate can be issued.

Azure Artifact Signing account

Pending. The subscription is ready, but the Artifact Signing account and Public Trust identity validation still need to be completed in the Azure portal.

Apple Developer enrollment

Pending. Enrollment in the Apple Developer Program is required before macOS Developer ID certificates, signing, and notarization can be completed.

Expected timeline

  1. 1

    Windows signed installer

    Blocked on Azure identity validation. Once validation passes and a Public Trust certificate profile is created, the CI workflow can produce a signed Windows installer. Identity validation typically takes minutes to hours, but it requires a mobile device for document and selfie capture.

  2. 2

    macOS signed and notarized installer

    Blocked on Apple Developer Program enrollment. After enrollment, Developer ID Application and Installer certificates must be created, exported, and added to repository secrets; then CI can sign, notarize, and staple the macOS build. Enrollment review usually takes 1–5 business days for individuals and 1–4 weeks for organizations.

  3. 3

    General consumer release

    Once both signed installers are available, the public download page will switch to the signed builds and the launch will be considered ready for general consumers.

What you can do today

Until signed installers are ready, you have two paths:

  • Wait for the signed build: Subscribe to release notifications or check back on this page. The download page will be updated as soon as signed Windows and macOS installers are available.
  • Use current development builds: Unsigned builds are published automatically from GitHub Actions. These are suitable for testing and development, but your operating system may show a security warning because the binaries are not yet signed. Only download builds from the official repository.

Do not bypass security warnings for installers from sources you do not trust.
