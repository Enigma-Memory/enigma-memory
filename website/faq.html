FAQ

Questions people ask before installing Enigma Memory.

Short answers for privacy, setup, AI app connections, proofs, and developer options.

Before you install

Do I need Node or npm?

No for the consumer desktop app. Node and npm belong to the Developer CLI appendix.

What platforms are supported?

macOS, Windows, and Linux. The desktop app bundles its runtime. See the download page for platform details.

Are the installers signed today?

Not yet for general public launch. Windows signing is blocked on Azure Artifact Signing/Public Trust completion, and macOS distribution is blocked on Developer ID signing, notarization, and stapling. See launch status before installing.

Privacy and control

Where is my vault?

In an Enigma-controlled local app location shown inside the desktop app. Public docs do not print machine-specific locations.

Does Enigma control AI provider records?

No. Enigma controls its local vault and the context it offers. Anything a provider stores must be checked through that provider's own controls and evidence.

Can I remove Enigma from a connected app?

Yes. Use the Connections screen in Enigma to disconnect a client, then follow the safe removal guide if you want to uninstall the desktop app. Removing Enigma from the client does not delete the provider's own logs or caches.

Connecting AI apps

Which AI clients are supported?

Claude Desktop, Cursor, Kimi Code, VS Code/Cline, Roo Code, OpenCode, and generic MCP clients. See Connect apps.

Do I have to edit app settings by hand?

No. The desktop app detects installed clients, previews the change, and applies it after you approve. Manual settings are advanced developer and troubleshooting material only.

Do I need to restart the AI client?

Usually yes. Enigma will tell you when a restart or window reload is needed after connecting.

Proofs

What can proofs show?

Proofs can show hashes, roots, opaque refs, counts, timestamps, signatures, and validation results for Enigma-controlled local events. See Proofs.

What can I safely share with support?

Health status, app version, sanitized error codes, and public-safe proof refs only. Keep private memory content, conversation text, sign-in material, provider identifiers, and local locations out of support messages.

Does Solana-ready mean submitted to a chain?

No. Solana-ready means prepared for review. Any on-chain transaction requires separate transaction evidence.

Troubleshooting

My installer is blocked. What do I do?

See Troubleshooting and Launch status for signing-safe guidance. Do not bypass warnings for installers from unknown sources.

Enigma says my vault is not ready.

Run Repair vault from the health dashboard first. If it still is not ready, open the Vault not ready guide for Restore from backup, Safe reset, and support-safe sharing guidance. CLI repair stays in the advanced troubleshooting panel.

Developers

Can I use Enigma from the command line?

Yes. The CLI appendix covers npm install, setup, doctor, manual MCP clients, proof export, and source checkout.

Can I build from source?

Yes. Source checkout is documented in the CLI appendix. Use it for package development, source-only docs, or advanced deployments.
