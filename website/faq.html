FAQ

Questions people ask before installing Enigma Memory.

Short answers for privacy, setup, AI app connections, proofs, and developer options.

Before you install

Do I need Node or npm?

No for the consumer desktop app. Node and npm belong to the Developer CLI appendix.

What platforms are supported?

macOS, Windows, and Linux. The desktop app bundles its runtime. See the download page for platform details.

Will the installer be signed?

Yes. Public launch requires a signed Windows installer or Store/MSIX path and a macOS Developer ID signed, notarized, and stapled installer. Until those steps are complete, you may see a security warning.

Privacy and control

Where is my vault?

In an Enigma-controlled local app location shown inside the desktop app. Public docs do not print local absolute paths.

Does Enigma delete memory from AI providers?

No. Enigma controls its local vault and context selection; provider deletion requires provider evidence.

Can I remove Enigma from a connected app?

Yes. Use the Connections screen in Enigma to disconnect a client, or remove the Enigma MCP entry from that client's settings. Removing Enigma from the client does not delete the provider's own logs or caches.

Connecting AI apps

Which AI clients are supported?

Claude Desktop, Cursor, Kimi Code, VS Code/Cline, Roo Code, OpenCode, and generic MCP clients. See Connect apps.

Do I have to edit MCP JSON?

No. The desktop app detects installed clients, previews the change, and applies it after you approve. Manual MCP JSON is only in the advanced panel.

Do I need to restart the AI client?

Usually yes. Enigma will tell you when a restart or window reload is needed after connecting.

Proofs

What can proofs show?

Proofs can show hashes, roots, opaque refs, counts, timestamps, signatures, and validation results for Enigma-controlled local events. See Proofs.

What can I safely share with support?

Health status, app version, sanitized error codes, and public-safe proof refs only. Do not send raw memory, prompts, transcripts, credentials, account IDs, or local paths.

Does Solana-ready mean on-chain?

No. Solana-ready means prepared for review. On-chain submission requires separate transaction evidence.

Troubleshooting

My installer is blocked. What do I do?

See Troubleshooting for signing-safe guidance. Do not bypass warnings for installers from unknown sources.

Enigma says my vault is not ready.

Run the in-app vault repair from the health dashboard. If that does not work, see the CLI repair commands in the CLI appendix.

Developers

Can I use Enigma from the command line?

Yes. The CLI appendix covers npm install, setup, doctor, manual MCP clients, proof export, and source checkout.

Can I build from source?

Yes. Source checkout is documented in the CLI appendix. Use it for package development, source-only docs, or advanced deployments.
