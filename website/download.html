When you have an installer

1. Open Enigma Memory

Use a signed desktop app when available. If you intentionally test an unsigned build, confirm this page labeled it testing-only before install.

2. Create your Memory Drive

The app creates a private local vault, resumes the wizard if you close it, and keeps paths out of the public support view.

3. Connect and test an app

Pick an AI app, preview what Enigma will change, test the local connection, and restart the app only when Enigma asks.

4. Import and review proof

Paste starter memories, approve a local write, then check proof counts without exposing raw text.

About signing and security warnings

Signed Windows and macOS installers are not ready yet. Until launch status shows signed installers, current desktop builds must be treated as unsigned testing and development builds.

If you intentionally test an unsigned build and your operating system warns before launch:

  • macOS: Only continue if you meant to test an Enigma development build from an official Enigma distribution channel.
  • Windows: Only continue if you meant to test an Enigma development build from an official Enigma distribution channel.
  • Linux: Verify the package checksum or signature through your distribution tools when one is provided.

Do not bypass security warnings for installers from unknown sources. See the current launch status for code-signing progress before recommending Enigma to a general consumer.
