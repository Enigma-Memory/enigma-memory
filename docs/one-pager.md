# Enigma one-pager

## What it is

Enigma is the portable memory and proof layer for AI. It lets people and enterprises own durable AI memory outside model providers and use it across ChatGPT, Claude, Kimi, Cursor, local models, and agent frameworks.

## Why now

AI memory is becoming valuable, but it is trapped inside subscriptions and vendor stacks. Enterprises need persistent memory for useful agents but cannot accept opaque retention, deletion, residency, or audit behavior. Developers have memory libraries, but not a universal custody and proof layer.

## What Enigma provides

- Memory Passport: portable user-owned memory.
- Vault: encrypted canonical memory event store.
- Capsules: shareable encrypted memory bundles.
- Receipts: signed lifecycle and boundary proofs.
- Verifier: offline validation with no server trust.
- Bridge: MCP, browser, API, framework, and enterprise gateway integrations.
- Mesh: decentralized encrypted sync and optional public root anchoring.

## Why it is different

Enigma is not another vector database. It treats vector search, summaries, and graph indexes as derived caches. The source of truth is a signed memory event stream with proof artifacts.

## First demo

Import memory from ChatGPT and Claude, use it in Kimi Code through MCP, delete a sensitive memory, and verify offline that Enigma no longer serves it.

## Claim boundary

Enigma proves Enigma-controlled memory state and declared boundary operations. It does not claim closed providers physically deleted data or that model weights forgot.
